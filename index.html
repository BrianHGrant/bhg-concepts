MY WORK

I like data.

Technology is expanding the amount of information we collect and have access to everyday. How to turn this raw information into actionable datasets is a question which interests me.

I recently worked on a project to create a heat map for reported bikes stolen in a specified city. This Node.js based application collects data from the Bike Index API via an AJAX GET request.
This information was then sent by POST to Google Maps API to geocode the addresses and create the map.

Currently I have been diving into the D3.js framework to look at other ways to visualize data.

I can monetize.

While developing apps is fun, we all need to make a living. Finding ways to spread the word about your application as well as make money is a part of a developer's job.

Here I made a survey app for users to enter a series of questions. Based on the user's answers a potential vacation spot would be suggested.

While this application began as a school project I took it the extra step with social media integration. Users can share their results as status updates to Facebook. I connected the application with Audience network to allow for advertisement based income potential when users visit the page through mobile devices.

I know Ruby.

Ruby is a great language. Whether using Sinatra to spin up a program in development or working on a full Rails deployment, I am learning all I can about the language and integrating databases into my applications using Active Record and other ORMs.

Click the photo to run a small program which uses a flat text file to verify if a word entered by user is in the English language. It will then give the user the Scrabble score. This program has been deployed on Heroku.

To see a more in-depth project, follow the link below to view the source code for a group project which accessed the Spotify API. This program allows for a group of people to contribute to a shared playlist. I worked on the encryption scheme which was used to securely store user passwords in a PostGreSQL database using DataMapper and Warden.

I like time.


I worked as part of a collaborative effort to create a Drum Machine in JQuery. This project presented some interesting challenges in managing time. For modern web developers, being able to manage asynchronous events is an important skill and this project gave me a greater understanding of time in JavaScript.

My role in this project was as a project manager as well as the backend logic for the looping script. Our team accomplished a lot. It was a challenge to create a editable loop with an interrupt in a single-thread language.

I like challenges.

What attracts me to software development is that no two applications are the same. Even if you are solved the problem before, there might be a different way to solve it. Maybe this way did not exist a few years before.

Right now in addition to my full-time studies in JavaScript and Ruby on Rails I am experimenting in Clojure, C#, and Java. I love to solve algorithms and have competed in some competitive coding challenges.
